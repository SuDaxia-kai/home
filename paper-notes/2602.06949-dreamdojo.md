论文题目：DreamDojo: A Generalist Robot World Model from Large-Scale Human Videos
作者：Shenyuan Gao 等
工作单位（optional)：NVIDIA, HKUST, UC Berkeley, UW, Stanford, KAIST, UofT, UCSD, UT Austin 等
发表时间：2026-02-06（arXiv:2602.06949v1）
论文链接：https://arxiv.org/pdf/2602.06949

## 研究问题
### 要解决什么问题？
在接触丰富（contact-rich）的机器人任务中，学习一个既动作可控、又能泛化到未见场景，并支持实时交互的 world model。

### 问题的数学描述
交互式 world model 的核心是学习状态转移分布：给定当前状态和动作，预测下一状态，可写为 $s_{t+1} \sim p(\cdot \mid s_t, a_t)$。

论文在 flow matching 目标上加入时间一致性项：
$$
\mathcal{L}_{\text{final}} = \mathcal{L}_{\text{flow}} + \lambda \mathcal{L}_{\text{temporal}}, \quad \lambda = 0.1
$$

### 研究内容的关键假设
- 人类与机器人交互中的底层物理规律具有可迁移性。
- 大规模人类视频虽然没有动作标签，但可通过连续 latent action 提供统一代理动作。
- 面向具体机器人平台时，仍需小规模目标机器人后训练做动作空间适配。

### 为什么重要？
真实机器人数据获取成本高、覆盖范围窄。若可利用大规模人类视频完成预训练并保持动作可控性，可显著提升 OOD 泛化与落地可用性。

## 技术方法
### 整个技术框架和原理（如适用）
整体采用 3 阶段：
- 人类视频预训练：In-lab + EgoDex + DreamDojo-HV（总计约 44,711 小时）
- 机器人后训练：在 G1, GR-1, AgiBot 等平台做动作空间适配
- 蒸馏部署：将 teacher 扩散模型蒸馏为实时自回归 student

### 系统架构图说明
论文包含 3 个核心网络模块。

- Latent Action Model（VAE + 时空 Transformer，约 700M）
输入：相邻帧（$f_t, f_{t+1}$）
输出：32 维连续 latent action
作用：从无标签视频提取可迁移动作语义

- World Model Teacher（基于 Cosmos-Predict2.5 初始化）
输入：条件帧 + 动作条件（latent action chunk 或机器人动作）
输出：未来视频帧
作用：学习动作条件下的未来预测

- World Model Student（蒸馏后因果注意力架构）
输入：历史上下文帧 + 动作
输出：自回归未来帧
作用：实时交互推理（文中报告约 10.81 FPS）

时间频率相关：后训练视频采样约 10 Hz，动作以 chunk 方式注入 latent 时间步。

### 具体算法（针对每个具体神经网络）
- Latent Action Model：24 层 encoder blocks + 24 层 decoder blocks。
- Teacher / Student：DiT 风格视频扩散骨干；student 使用因果注意力并减少采样步数（如 35 步到 4 步）。
- 推断：Teacher / Student 的逐层完整配置在主文未完全展开。

### 每个神经网络的架构
- Latent Action Model：输入相邻帧，输出 latent action，并重建后帧。
- Teacher：输入条件帧与动作，预测未来帧分布。
- Student：输入历史帧与动作，做快速自回归生成。

### 训练目的和 loss function
Latent Action Model 的目标：
$$
\mathcal{L}_{\text{pred}} = \mathbb{E}_{q}\left[\log p(f_{t+1} \mid \hat{a}, f_t)\right] - \beta D_{\mathrm{KL}}\left(q(\hat{a} \mid f_t:f_{t+1}) \Vert p(\hat{a})\right)
$$
其中 $\beta = 10^{-6}$。

World Model 预训练目标：$\mathcal{L}_{\text{final}} = \mathcal{L}_{\text{flow}} + \lambda \mathcal{L}_{\text{temporal}}$。

Student 蒸馏：先 warmup 回归 teacher ODE 轨迹，再通过 distillation 目标最小化 teacher 与 student 分布差异。

### 如何获取训练数据
- 人类视频：In-lab, EgoDex, DreamDojo-HV
- 机器人数据：Unitree G1, Fourier GR-1, AgiBot, YAM
- 预训练采样比：In-lab : EgoDex : DreamDojo-HV = 1 : 2 : 10

### 训练算法实践中的 insights 和 tricks
- 使用 relative action 表示以提升动作泛化。
- 使用 chunked action injection 对齐 latent 时间尺度。
- action MLP 最后一层零初始化，降低对预训练参数的扰动。
- 后训练阶段重置 action MLP 第一层以匹配目标动作空间。
- temporal consistency loss 对动作跟随与目标完整性均有增益。

## 实验结果
### 实验环境是什么，如何构建？
- 评测集：In-lab Eval, EgoDex Eval, EgoDex-novel Eval, DreamDojo-HV Eval, DreamDojo-HV-novel Eval, Counterfactual Eval
- 指标：PSNR, SSIM, LPIPS + 人类偏好评估
- 训练规模：预训练最高使用 256 × H100，后训练常用 128 × H100，蒸馏约 64 × H100

### 对比的 baseline 算法有哪些？
- Cosmos-Predict2.5（无人类视频预训练）
- action-free pretraining
- ground-truth action conditioning（retargeted action / MANO，理想上界）
- w/o pretrain vs w/ pretrain（蒸馏后泛化对比）

### 重要结果总结
- latent action 条件相比 action-free 与 w/o pretrain 更优，并接近理想动作标签上界。
- 增加多样化人类视频数据后，OOD 与 counterfactual 评测持续提升。
- 人类评估中 DreamDojo（尤其 14B）在物理正确性与动作跟随上优于 Cosmos-Predict2.5。
- 蒸馏后 student 约 10.81 FPS，明显高于 teacher（约 2.72 FPS），且质量下降有限。

## 总结
### 文章最主要的 idea 是什么？
通过大规模人类视频预训练 + 连续 latent action 代理动作，实现可控且可泛化的机器人 world model，并通过蒸馏获得实时性能。

### 最大的亮点是什么？
同时兼顾了动作可控性、OOD 泛化与实时推理这 3 个目标。

### 重要拓展方向？
- 强化跨 embodiment 对齐能力。
- 与策略学习深度结合，形成闭环 planning / control。
- 提升长时滚动稳定性与异常行为建模。

### 其它 critiques
- 训练成本高，复现门槛高。
- 仍需目标机器人后训练，完全零样本跨平台迁移有限。
- 部分细粒度结构超参数公开有限。
